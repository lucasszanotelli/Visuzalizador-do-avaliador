Guia de avaliacao automatica

Avaliacao das respostas do chatbot

Para iniciar a avaliacao das respostas geradas por um chatbot, foi necessario construir uma base de perguntas e respostas consideradas ideais, que serviram como referencia para comparacao. Para isso, utilizamos como base o FAQ disponivel no site oficial do CAR, no endereco https://www.car.gov.br/#/suporte.

Base de referencia e gabarito humano

Classificacao inicial antes das metricas.

Antes da aplicacao das metricas automaticas, cada resposta gerada pelo chatbot foi analisada manualmente por uma pessoa. Essa avaliacao humana serviu como gabarito da comparacao, classificando cada resposta como:

  • 1: resposta correta;
  • 0: resposta incorreta.

Essa etapa foi importante porque permitiu definir uma referencia confiavel para avaliar se os resultados calculados automaticamente pelo script estavam coerentes com a interpretacao humana.

Como foi feita a avaliacao automatica?

Comparacao entre respostas do FAQ e do chatbot.

As mesmas perguntas presentes na base do FAQ foram enviadas para a IA do chatbot. Em seguida, as respostas geradas pelo chatbot foram comparadas com as respostas oficiais da FAQ por meio de um script desenvolvido em Python. Esse script calcula valores entre 0 e 1 para dois criterios principais.

Criterio lexical

Avalia o quanto a resposta do chatbot utiliza palavras semelhantes ou iguais as da resposta considerada ideal. Esse criterio e importante porque, em alguns casos, determinadas palavras-chave sao essenciais para que a resposta seja considerada adequada.

Por exemplo, na pergunta “O que e CAR?”, espera-se que a resposta mencione termos como “Cadastro”, “Ambiental” e “Rural”.

Criterio semantico

Avalia o sentido da resposta, ou seja, se a resposta do chatbot transmite a mesma ideia da resposta oficial, mesmo utilizando palavras diferentes.

Esse criterio possui maior peso, pois uma resposta pode estar correta mesmo sem repetir exatamente os mesmos termos da FAQ.

Para calcular a nota final, foram atribuidos pesos diferentes para cada criterio: 80% para o criterio semantico e 20% para o criterio lexical.

score_final = (score_semantico * 0.8) + (score_lexical * 0.2)

Thresholds

Definicao de limites para classificar respostas.

Apos calcular o score final de cada resposta, surge uma questao importante: a partir de qual nota uma resposta pode ser considerada correta ou incorreta? Para isso, foram definidos os thresholds, que sao valores de corte utilizados para classificar as respostas. Um threshold define o limite minimo necessario para que uma resposta seja considerada correta. Respostas com score igual ou superior ao threshold sao classificadas como corretas, enquanto respostas com score inferior sao classificadas como incorretas.

Exemplo com score 0,65

  • considerada correta com threshold 0,60;
  • considerada incorreta com threshold 0,70.

Analise por intervalo

Esse processo foi repetido para diferentes valores de threshold, permitindo analisar o comportamento das metricas de avaliacao em cada intervalo. Dessa forma, torna-se possivel identificar qual valor de threshold apresenta o melhor equilibrio entre respostas corretas e incorretas.

Metricas de avaliacao

Accuracy, precision, recall e F1-score.

Para avaliar o desempenho do chatbot em cada threshold, foram utilizadas metricas classicas de classificacao. Antes de explicar cada metrica, e importante entender quatro conceitos principais.

VP — Verdadeiro Positivo

A resposta era correta e o sistema classificou como correta.

VN — Verdadeiro Negativo

A resposta era incorreta e o sistema classificou como incorreta.

FP — Falso Positivo

A resposta era incorreta, mas o sistema classificou como correta.

FN — Falso Negativo

A resposta era correta, mas o sistema classificou como incorreta.

Formulas e interpretacao

Como cada metrica e calculada.

Accuracy

Mede a proporcao total de classificacoes corretas feitas pelo sistema. Ela considera tanto as respostas corretas classificadas corretamente quanto as respostas incorretas classificadas corretamente. Em outras palavras, a acuracia mostra o percentual geral de acertos do modelo.

accuracy = (VP + VN) / (VP + VN + FP + FN)

Precision

Mede quantas das respostas classificadas como corretas realmente estavam corretas. Essa metrica e importante para verificar se o sistema esta sendo muito permissivo, ou seja, se esta classificando muitas respostas erradas como corretas. Quanto maior a precisao, menor a quantidade de respostas incorretas sendo aceitas como corretas.

precision = VP / (VP + FP)

Recall

Mede quantas das respostas que realmente estavam corretas foram identificadas corretamente pelo sistema. Quanto maior o recall, menor a quantidade de respostas corretas sendo classificadas como incorretas.

recall = VP / (VP + FN)

F1-score

Combina precision e recall em um unico valor. Ele e especialmente util quando queremos encontrar um equilibrio entre aceitar respostas corretas e evitar aceitar respostas incorretas. O F1-score e mais indicado quando existe um desequilibrio entre respostas corretas e incorretas na base de avaliacao, pois ele penaliza situacoes em que a precision ou o recall estao muito baixos.

f1 = 2 * (precision * recall) / (precision + recall)

Consideracoes finais

Equilibrio entre thresholds e metricas.

Com esse processo, e possivel comparar a avaliacao automatica do chatbot com a avaliacao humana, observando como cada threshold influencia as metricas de desempenho. Assim, a escolha do melhor threshold nao depende apenas do score final, mas tambem do equilibrio entre as metricas. Um threshold muito baixo pode aceitar respostas incompletas ou incorretas, enquanto um threshold muito alto pode rejeitar respostas que estao semanticamente corretas, mas escritas de forma diferente da FAQ.

Portanto, a analise dos thresholds permite definir um criterio mais confiavel para avaliar a qualidade das respostas geradas pelo chatbot.
